1. LLM Fundamentals

At its core, a Large Language Model is a statistical engine designed to predict the next token in a sequence. By processing trillions of words, it learns a "world model" that captures grammar, reasoning, and factual relationships.

graph TD A[Raw Data] --> B[Tokenization] B --> C[Embedding Space] C --> D[Transformer Layers] D --> E[Attention Mechanism] E --> F[Next Token Prediction] F --> G[Generated Response]

Core Vocabulary:

  • Parameters The numerical weights (often billions) that define the model's intelligence.
  • Tokens Sub-word units. "Signal" might be one token, while "Unrealized" might be three.
  • Inference The act of running a prompt through the model to get a result.
  • Context Window The "memory" of the current conversation (e.g., 8k, 32k, or 1M tokens).

2. The Transformer Architecture

Introduced in the 2017 paper "Attention Is All You Need," the Transformer architecture replaced sequential processing (like RNNs) with parallelizable attention mechanisms.

The Attention Mechanism

Self-attention allows the model to look at every word in a sentence simultaneously and decide which words are most relevant to the current word being processed. In the sentence "The bank was closed because it was a holiday," attention helps the model know that "it" refers to "the bank" and not "the holiday."

Simplified Attention Logic:
1. Query (Q): What am I looking for?
2. Key (K): What do I contain?
3. Value (V): What is my content?
Attention(Q, K, V) = softmax(QKᵀ / √dₖ)V

Training Phases:

  • Pre-training: Learning the general patterns of language from the internet.
  • Fine-tuning (SFT): Teaching the model to follow specific instructions (Chatting).
  • Alignment (RLHF): Using human feedback to ensure safety and helpfulness.

3. Local vs. Cloud Execution

Where a model runs dictates its speed, privacy, and cost. Modern AI development requires choosing between these two fundamentally different paradigms.

The Cloud Paradigm (API-Based)

Models like GPT-4 or Gemini 1.5 Pro are too large for consumer hardware. They run on thousands of H100 GPUs in data centers. Users connect via HTTPS and pay per million tokens.

The Local Paradigm (On-Premise)

Models like Llama 3 or Mistral are "quantized" (compressed) to fit on home PCs. Tools like LM Studio allow these models to run entirely offline, utilizing your local GPU's VRAM.
